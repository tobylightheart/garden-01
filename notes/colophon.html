Every garden should tell you how it grows.

The soil

Plain static HTML, one hand-written CSS file, and one small script. No framework, no bundler, no build step, no dependencies. To work on it you open a file in a browser; to publish it you copy the files to a server. That's the whole pipeline.

  • HTML — one page per note, written by hand.
  • styles/garden.css — the look. Botanical palette, dashed lines, a leaf that sways. Uses CSS custom properties and color-mix() so the two seasons share one set of colours.
  • scripts/garden.js — ~30 lines. Only job: flip between day and dusk and remember which you chose. Without it, the garden still works — it's just always day.

The seasons

Tap ☀️ in the corner to switch between day and dusk. Your choice is saved locally; on a first visit the garden quietly follows your system's light/dark preference.

The plants that grow nowhere else

A few small touches I'm fond of:

  • The favicon is an inline SVG seedling — no extra file to fetch.
  • Internal links sprout a little ❧ leaf before them.
  • The title leaf sways, but holds still if you've asked your system to reduce motion.
  • Every note wears its ripeness honestly: 🌱 🌿 🌳.
